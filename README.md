# Study Planner Bot

Telegram-бот для організації навчального процесу. Допомагає створювати плани, додавати завдання, відстежувати дедлайни та отримувати нагадування.

## Можливості

- 📚 **Створення навчальних планів** - групуйте завдання за предметами
- ➕ **Додавання завдань** - додавайте завдання до планів з дедлайнами
- 📋 **Перегляд планів** - переглядайте всі свої плани та завдання
- 🔔 **Нагадування** - автоматичні нагадування за 24 години, 1 день та 1 тиждень до дедлайну
- ✅ **Відстеження прогресу** - позначайте виконані завдання

## Технології

- **Python 3.13+**
- [python-telegram-bot](https://github.com/python-telegram-bot/python-telegram-bot) - Telegram Bot API
- **SQLite** - локальна база даних
- **asyncio** - асинхронне виконання

## Встановлення

1. Клонуйте репозиторій
2. Створіть віртуальне середовище:

```bash
python -m venv .venv
.venv\Scripts\activate  # Windows
```

3. Встановіть залежності:

```bash
pip install -r requirements.txt
```

4. Створіть файл `.env`:

```env
TOKEN=your_telegram_bot_token
DATABASE_URL=sqlite:///study_planner.db
REMINDER_1_DAY=86400
```

## Запуск

```bash
python main.py
```

База даних `study_planner.db` створюється автоматично при першому запуску.

## Використання

### Команди

- `/start` - Головне меню
- `/help` - Допомога

### Головне меню

Бот пропонує 4 кнопки для навігації:

- 📚 **Новий план** - створити новий навчальний план
- ➕ **Додати завдання** - додати завдання до існуючого плану
- 📋 **Мої плани** - переглянути всі ваші плани та завдання
- 🔔 **Нагадування** - переглянути активні нагадування

### Створення плану

1. Натисніть "📚 Новий план"
2. Введіть назву предмета
3. План створено!

### Додавання завдання

1. Натисніть "➕ Додати завдання"
2. Оберіть план
3. Введіть назву завдання
4. Оберіть дату дедлайну через календар

### Робота із завданнями

У плані натисніть на завдання, щоб:
- ✅ Позначити як виконане
- ↩️ Позначити як невиконане
- 🗑️ Видалити завдання

## Архітектура

### Сервіси

- **StudyService** - створення/отримання планів і завдань
- **ReminderService** - планування та надсилання нагадувань

### Обробники

- **StartHandler** - команда `/start`, реєстрація користувача
- **HelpHandler** - команда `/help`
- **CallbackHandler** - всі натискання інлайн-кнопок
- **UserMessageHandler** - текстові повідомлення

### Сховище

**SQLiteStorage** реалізує інтерфейс `StorageInterface` та забезпечує:
- Збереження користувачів, планів, завдань
- Нагадування в базі даних
- CRUD операції

### Схема бази даних

```sql
users (user_id, username, timezone)
plans (plan_id, user_id, subject, deadline)
tasks (task_id, plan_id, title, deadline, is_done)
plan_tasks (plan_id, task_id)
reminders (task_id, user_id, title, deadline, reminder_type, delta_seconds, sent)
```

## Система нагадувань

1. При створенні завдання `ReminderService.schedule_task_reminders()` створює нагадування
2. Фоновий процес запускається кожні 60 секунд
3. Перевіряє нагадування, термін яких настав
4. Надсилає повідомлення через Telegram Bot API
5. Позначає нагадування як надіслане

### Типи нагадувань

- **24годин** - за 24 години до дедлайну (якщо дедлайн >= 1 день)
- **завтра** - за 1 день до дедлайну
- **тиждень** - за 1 тиждень до дедлайну (якщо дедлайн >= 7 днів)
